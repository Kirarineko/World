# 计算机科学学部

> 简称：计科。HBI 三个学部中年纪最大的，也是建校时最初的两个支柱学科之一（与海洋科学并列）。

## 学科定位

计科在 HBI 的定位与综合性大学的计算机系有所不同：它不追求面面俱到，而是集中力量在两个方向：**高性能计算**和**生物信息学**。前者是工具，后者是应用。除此之外，非线性系统建模（包括魔力流模拟）是近年新增的第三个支点。

## 核心方向

### 一、高性能计算

| 条目 | 说明 |
|------|------|
| 硬件 | 校内有一座小型数据中心（约 200 个节点），托管在 C4 实验楼。算力在全国高校中排不进前三十，但在生物信息学这个细分领域够用 |
| 主要负载 | 蛋白质折叠模拟、菌丝体信号网络数据分析、神经元集群建模：都是生科的课题 |
| 特色 | 不追求单机算力，侧重分布式任务调度。C4 有一套自研的调度系统"潮汐"：根据实验楼群各实验室的实时负载动态分配计算资源。名字来自"海洋科学的人觉得像潮汐涨落" |

> 数据中心冷却系统的废热在冬季用于加热邻近的两栋宿舍楼。学生管这叫"冬季暖气由蛋白质折叠提供"。

### 二、生物信息学

| 条目 | 说明 |
|------|------|
| 研究对象 | 从生科和海科涌入的序列数据：藻类基因组、菌丝体信号转录组、环境 DNA 样本等 |
| 工具链 | 自主维护了一套针对非模式生物的基因组分析管线。非模式生物的意思是"不是小白鼠、果蝇、拟南芥那种大家都用的物种"。在 HBI，最常见的样本来源是海湾里的某只贝类和保留林地里的某种野生菌 |
| 与海科交叉 | 环境 DNA 项目：从一升海水中提取的 DNA 片段可以识别其中存在哪些物种，不需捕捞。计科负责序列拼接和物种注释，海科负责取样和验证 |

### 三、非线性系统建模与魔力流模拟

这是计科近年最年轻的课题，也与生科的魔力-神经接口课题有深度交叉。

| 条目 | 说明 |
|------|------|
| 研究内容 | 魔力在介质（空气、水体、活体组织）中的传播不是线性的。它受环境魔力浓度、介质密度、附近魔力源强度等多重因素耦合影响。计科的目标是建立可计算的魔力流体力学模型 |
| 进展 | 已有二维平面模型可较准地模拟实验室内可控环境下的魔力扩散。三维自由环境模型仍在开发中（复杂度过高，一次模拟就能吃空 C4 的全部算力半天 |
| 应用 | 目前主要是科研用途）帮助生科在实验前预判魔力场的分布。如果有朝一日模型成熟，理论上可以预测某区域内任意一点的即时魔力强度：这在工程层面意味着可以优化魔力储能设备的灌魔效率 |

### 四、神经网络建模

| 条目 | 说明 |
|------|------|
| 方向 | 不限于人工神经网络。也包括对生物神经网络的建模仿真：湿件计算机需要一个计算模型来描述"一群神经元在一起到底在算些什么" |
| 与生科合作 | 生科提供湿件的电信号数据集，计科负责分析其中的信息编码方式。目前已知的是：湿件神经元的放电时序模式比同等规模的人工神经网络丰富约一个数量级，因为生物神经元有疲劳、有随机放电、有化学调制。这些"噪声"在人工网络中是需要消除的缺陷，在湿件中可能是表达能力的一部分 |

## 教学特色

| 条目 | 说明 |
|------|------|
| 强制跨学科 | 计科本科生从大二起必须修一门生科或海科的实验课。反之亦然。这门课对计科学生的打击通常是毁灭性的：他们第一次意识到自己的手在培养皿面前有多不稳，而隔壁生科学生单手移液的同时还在刷手机 |
| 毕业设计 | 计科学生不能做纯软件课题。至少需要一个与物理世界交互的组件：传感器、机器人、生物接口，任选其一。这条规定是某位前系主任留下的，据说他的原话是"写了四年代码毕业了还不敢摸电线，那是技校不是大学" |

## 学部氛围

计科楼是全校最安静的建筑，不是因为学生不交流，是因为所有人都戴耳塞。和生科楼潮湿的泥土味、海科楼微咸的海水味不同，计科楼闻起来是热的电子元件和冷却液。

计科学生有两个群（一个在社交软件上，讨论作业和组队；另一个在 C4 的本地聊天服务器上）那台服务器因为某次安全演练被封闭在一个内部网络中没恢复，意外成了计科内部的"暗网"。"暗网"上没有学号没有姓名只有昵称，讨论范围从服务器维护到校园八卦到魔力流模型的最新论文。学校知道它的存在，但不知道入口，因为入口每学期换一次。

## 速查

```
高性能计算 → 200节点小型数据中心，"潮汐"调度系统
生物信息学 → 非模式生物基因组分析，环境DNA
魔力流模拟 → 二维模型可用，三维还在跑
神经网络建模 → 人工+生物双线，湿件信号分析
硬核传统 → 计科生必须修实验课，毕设必须有物理交互组件
内部"暗网" → C4本地聊天服务器，学校知道存在但找不到入口
```
