# 雷云科技（Raiun）

> 它不造模型的时候。它造的是造模型的工厂。

## 概况

| 条目 | 说明 |
| - | - |
| 名称 | 雷云科技 |
| 英文 | Raiun |
| 成立 | 约 24 年前（公开日同年） |
| 员工 | 约 1200 人 |
| 总部 | 星月湾月心区，数据中心设于北侧山丘后方 |
| 业务 | 云服务（CPU/GPU/湿件）、云应用、数据库、模型推理 API |
| 定价 | 全部按 token 计费，部分模型免费 |


雷云科技是星月湾最大的云服务与 AI 企业。它的成立与公开日几乎同期

二十四年后，它已经从一家算力转售商成长为拥有自有通用大模型、自有开发者工具生态的完整 AI 企业。

## 计算产品线

Raiun 的算力产品分为三个层级：按性能、延迟和成本梯度排列，覆盖从"即时要"到"慢慢跑"的全部场景。

| 产品 | 硬件 | 延迟 | 成本 | 适用场景 |
| - | - | - | - | - |
| **钢云** | CPU 集群 | 毫秒级 | 基准 | Web 服务、数据库、批处理 |
| **赤云** | GPU 集群（含 S200） | 毫秒级 | 高 | 低延迟推理、模型微调、实时交互 |
| **髓云** | 湿件集群（Genius2 衍生） | 数十至百毫秒 | 极低（~5%） | 高吞吐推理、无需实时响应的批量任务、研究者调试 |


### 钢云

常规计算。Raiun 的钢云产品线本质上是一个现代化的 VPS 与数据库托管平台：虚拟服务器、对象存储、托管数据库（关系型与非关系型）、消息队列。B 端客户用它跑 Web 后端和数据分析；C 端客户用它挂个人博客和云盘同步。

### 赤云

GPU 算力。赤云的核心硬件是研究园区 S200 加速卡及同类 GPU：48GB HBM，312 TFLOPS。赤云提供裸金属 GPU 租赁和托管的推理即服务两种模式：

- 裸金属：客户直接获取 GPU 节点的 root 权限，自部署模型和推理框架。研究园区自身在扩容需求超出园区算力上限时也会通过赤云租赁额外节点

- 推理即服务：Raiun 托管推理基础设施，客户通过 API 调用。Thunder 和 Rain 即由此线路交付

### 髓云

这是 Raiun 最具星月湾特色的产品：湿件计算即服务。

髓云的硬件基底是 Genius2 皮层类器官阵列：每个类器官含约 850 万个活体人类皮层神经元，经 3D MEA 探针（8192 通道）连接至硅基接口。单次推理延迟在 30–80ms 之间，远超硅基替代方案，但单位 token 的能耗仅为赤云的约 5%。

髓云适用于那些不在乎多等几十毫秒但需要极高吞吐量的任务：大规模文本分类、离线知识图谱构建、夜间批处理推理。它也是全泡最便宜的推理选项：一个研究生用每月的免费配额就够跑完大半篇论文的实验。

> 髓云的名字不只关于骨髓。本地工程师中流传另一个解读：它像潮髓：无数简单的个体连成一张网，从无意识的群体中涌出可以思考的东西。

## 自研模型

Raiun 不满足于托管别人的模型。它自研了两条通用大模型线路：Thunder 和 Rain。两者均与星澜 Ripple 系列构成竞争关系。Raiun 的模型团队和 HBI/研究园区之间没有正式合作：互引论文，互测 benchmark，互为压力。

### Thunder

**编程特化通用 LLM。**

Thunder 的核心能力集中在代码生成、代码理解、程序修复和自然语言转结构化查询。它不是只懂 Python：覆盖主流编程语言及部分科学计算领域语言。在代码基准上，Thunder 的多项指标强于星澜 Ripple。

Thunder 的内部分工包含两条子路线：一条追求代码生成的绝对精度（用于自动化 CI/CD 流水线和智能合约审计），一条追求与开发者的交互流畅度（用于 RaiunCoder）。

### Rain

**多模态通用 LLM。**

Rain 的输入模态覆盖自然语言、图像、视频帧和结构化数据表。输出可以是文本描述、分类标注、跨模态检索结果，或者基于视觉输入生成的代码片段。在多模态基准上，Rain 的多项指标强于星澜 Ripple。

Rain 的典型应用包括（医疗影像预筛（与星月湾人民医院有非正式合作，影像科的部分初筛流程使用 Rain 预标注）、遥感图像自动解译（某个不愿透露名称的客户）可能是海洋大学，也可能是恒动：每月调用大量 token）、以及跨模态代码生成（"从这张 UI 设计稿写前端代码"）。

## RaiunCoder

\*\*基于 Thunder 的编程辅助工具。\*\*RaiunCoder 是 Raiun 旗下知名度最高的产品：知名度甚至超过公司本身。许多开发者知道"那个打完注释自动补全函数的猫娘图标插件"，但不知道它背后是一家叫 Raiun 的公司的模型在跑。

- **CLI 版本**：终端原生，可管道集成。自动补全、代码搜索、shell 命令辅助。支持本地模型（开发者自部署 Thunder 轻量版本）或云端 API 连接

- **GUI 版本**：IDE 插件形态（支持主流编辑器）。侧边栏对话式代码助手、行内补全、重构建议、测试用例生成

RaiunCoder 的免费层配额足够个人开发者日常使用：这是 Raiun 获客策略的核心。大学生用 RaiunCoder 免费层写完毕业设计，入职后说服公司购买企业许可证。路径早已被踩熟。

> RaiunCoder 的设置面板里有一个滑块：本地推断 / 混合 / 云端。滑块拉到最左，所有推理在你的电脑上跑，不需要网络。拉到最右，云端全速响应。中间是默认：IDE 会在你打字快的时候自动切到本地，在复杂重构请求时自动切到云端。这个滑块的名字叫"自知之明"。

## 开源模型托管

Raiun 的 API 平台上托管了多种第三方开源大模型的推理接口：客户不需要自己部署和调优，直接调 API。托管列表持续更新，随社区新模型发布而跟进。

免费层（每个账户每日可获得一定数量的免费 token 配额）可以在多个被托管模型之间自由分配。配额耗尽后按标准费率计费。免费配额由企业自行承担算力成本：本质上是品牌投资。一个今天在免费层挣扎的学生可能就是明天签约企业许可的架构师。

## 定价

全部按 token 计费，不分 B/C 端。核心原则：赤云贵在低延迟，髓云便宜在低能耗，钢云夹在中间。不同模型因推理算力需求差异（Thunder 大型版本的小时级成本远高于开源轻量模型）按不同 token 单价计费。

免费模型清单定期更新。免费不代表低质：Raiun 有意将部分自研模型的轻量版本放入免费层。不是慷慨。是试吃。

## 基础设施

Raiun 的数据中心选址于星月湾北侧山丘后方：严格避开了海湾的直接盐雾侵蚀范围。山丘本身构成了对台风季节最强风向的屏障。数据中心冷却系统利用了山丘北侧常年较低的地下水温度，不是水冷，是水泵循环的地下冷源。

湿件集群（Genius2 类器官）的维护是单独维度的工程挑战。类器官每 4–6 个月更换一次：替换周期由髓芯计算团队的生物监控系统自动管理。旧类器官的处置遵循 HBI 生命科技学部制定的生物伦理流程。Raiun 不是唯一使用湿件的企业，但它是唯一一家把湿件写进产品定价页的公司。

## 城市关系

Raiun 的客户遍及全泡，但它的根在星月湾。

这是一个充分内循环的城市：研究园区出硬件，HBI 出基础研究，雷云出商业化的算力包装和 API

但 Raiun 并不依赖这座城市。Thunder 和 Rain 的客户遍布大陆。RaiunCoder 的用户可能在另一个半球。髓云的湿件集群在电力最便宜的时段自动切换至离线批处理：处理的是远在数千公里外的客户提交的隔夜任务。

> Raiun 的第一间办公室在月心区一栋三层老楼的二层：当时只有十二个工位和一台从研究园区淘来的二手 S200 加速卡。二十四年后，老楼还在。十二个工位被保存成"雷云博物馆"（进门第一个展品是那台加速卡，旁边的标签写着）这是我们借来的第一个火花。后来我们还了。
