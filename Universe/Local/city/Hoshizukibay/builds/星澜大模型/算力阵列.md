# 异构算力阵列：S200 + Genius2

> 红伟达 S200（硅基 GPU 阵列）+ Genius2 生物神经元阵列（湿件协处理器）
> 代号：Half-Moon Bay Array

## 红伟达 S200

红伟达（HongWeiDa）是本地泡图形与加速计算领域的主导厂商。S200 是其面向 AI 训练/推理的旗舰 GPU，大致对标 2020 年代初的行业主流架构——不像后世的万亿晶体管怪物，但在合理的功耗墙内做到了足够大的显存带宽。

| 条目 | 规格 |
|------|------|
| 型号 | 红伟达 S200 |
| 架构 | Aegis-5，12nm 制程 |
| 单卡 FP16 算力 | 约 312 TFLOPS（Tensor Core） |
| 单卡显存 | 48 GB HBM2e |
| 互联 | 红伟达专有 NV-Link 3.0，双向 600 GB/s |
| 星澜阵列规模 | 128 张 S200，分装于 16 台 8-GPU 机柜 |
| 阵列总显存 | 6 TB（聚合），18 TB（含模型并行冗余） |
| 阵列总 FP16 | 约 40 PFLOPS |
| 采购 | 一半购入，一半由红伟达高校合作计划捐赠 |
| 散热 | 风冷为主，冬季废热接入邻近宿舍供暖 |

S200 在星澜中的角色是"快速脑"——负责语言表层生成、注意力计算、Token 概率采样。它在接受到输入请求后约半秒内产出一份流畅但保守的答复草案，这份草案随后被传递到 Genius2 入口缓冲区。

## Genius2：生物神经元协处理器

Genius2 是 HBI 生命科技学部湿件计算方向的第三代实验阵列——严格来说它不算一个"产品"，更像是一组活体实验平台在特定项目中被临时赋予了功能角色。

| 条目 | 规格 |
|------|------|
| 代号 | Genius2（第二代相对稳定培养方案，Genius1 全死在两周内了） |
| 基本单元 | 一个标准培养单元 = 6×6 cm² 微电极阵列（MEA，64×64 = 4,096 个电极位点）上培养的大鼠海马神经元单层网络 |
| 单元数量 | 64 个（8×8 排列，分两架，每架四层） |
| 单个单元神经元估算 | ~200,000–300,000 个 |
| 阵列总神经元 | 约 1,500 万 – 2,000 万个 |
| 培养基 | 改良 Neurobasal 培养基，含微量的魔力敏化添加剂——一种从星月湾海藻中提取的硫酸多糖的衍生物。不确定加了它有什么效果，但湿件在放大器上的信噪比确实比不加高约 15% |
| 信号读出 | 64×64 电极，每单元每秒采集 4,096 通道电压信号，采样率 20 kHz |
| 信号输入（刺激） | 同样通过 MEA 电极位点施加电刺激脉冲，诱发神经网络响应 |
| 培养单元生命周期 | 约 3–4 周。到期后神经元活性不可逆下降，需更换 |
| 更换周期 | 每批 64 单元分两组错峰更换（每组 32 单元），以维持阵列在更换期间仍有部分算力在线 |
| 温控 | 36.8 ± 0.2°C，独立冗余制冷。曾单次失控至 ~41°C，停摆三小时后恢复 |
| 魔力屏蔽 | 每个培养单元外覆 2mm 铝盒（魔力屏蔽标准方案），同时屏蔽外部电磁干扰和残留魔力背景场 |

### 培养单元维护

培养单元的日常维护由生科学部一名专职技术员负责——此人的正式职位是"神经组织培养与电生理监测技师"，但星澜项目组的人叫他/她"园长"。每批新培养单元上线时，"园长"会在其中一个单元的培养皿外壁贴一小张便签，上面是手写的上线日期和一句随想。旧便签贴在机房内侧墙面上，三年下来积累了大约四十张。

> 贴在 2024 年梅雨季的那张写的是："这批神经元的源鼠是三年来最暴躁的一只。期待。"

### 湿件与硅基的通信协议

S200 与 Genius2 之间不能直接通信——一端是 PCIe 总线上的数据包，另一端是培养皿里的电压波形。转换由一套定制中间件完成，内部称"半月湾中介层"（Half-Moon Bay Interposer）：

```
用户请求
    ↓
S200  → 生成答复草案（<0.5s）
    ↓
草案文本 → 编码为稀疏刺激模式（刺激哪些电极、什么频率、什么序列）
    ↓
Genius2 阵列 → 受刺激后自由演化 ~1–3 秒 → MEA 读出响应波形
    ↓
响应波形 → 解码为"推演修正向量" → 叠加到 S200 草案的 Token 概率分布上
    ↓
S200 → 应用修正向量后重新采样 → 输出最终 Token
    ↓
循环直到生成完整回复
```

关键点：Genius2 不生成文本。它不"输出一个 token"。它在看到 S200 的草案后，对这个草案的 Token 概率分布施加一个微弱的偏移——某个词多一点可能性，某个词少一点可能性。这个偏移量始终很小（通常 < 2%），但当它在多个连续的生成步骤中持续指向同一个方向时，最终输出的文本就会与 S200 的纯硅基草案出现可感知的差异。

研究人员描述这种感觉是"像有人在方向盘上轻轻搭了一只手"。

### Genius2 的慢性问题

| 问题 | 现象 | 对策 |
|------|------|------|
| 信号漂移 | 同一培养单元在同一刺激模式下，不同日期的响应波形逐渐偏移。一个月大的培养单元和一周大的培养单元面对相同输入时的输出不再可互换 | 无可行的工程对策。目前的做法是每天重校准，把当天的响应归一化到统计基线 |
| 单元间不可互换 | 两个同批次、同源、同培养条件的单元，对同一刺激的响应可能显著不同。不存在两个行为完全一致的 Genius2 单元 | 每个单元独立校准，向阵列输入刺激时实际上在向 64 个独立的"专家"同时请教 |
| 沉默单元 | 每个批次总有 1–3 个单元几乎不对任何刺激做出超出噪声水平的响应——神经元还活着（能测到基础放电），但似乎"不参与"。原因不明，可能是 MEA 与神经元的耦合质量偶然极差 | 标记为静默，不参与推理——训练期间静默单元被电压自检排除在信号采集范围外 |
| 死亡单元 | 个别培养单元内的神经元在生命周期内提前衰亡。信号强度陡降至噪声水平以下 | 隔离并更换。不影响阵列其余部分 |

> 计科楼内部一次项目组闲聊中，有个研究生对着一个被标记为"沉默"的单元看了很久，说："别人都在忙，它在发呆，有点像人类。"项目主管路过听见了，说："它没有像人类。你在楼里待太久了，出去晒会太阳。"但没人知道项目主管后来把这句话写进了个人周报。

## 速查

```
S200       → 128张, 16机柜, ~40 PFLOPS FP16, 6 TB聚合显存
Genius2    → 64培养单元, ~2000万神经元, 半月湾中介层桥接硅基与湿件
推理机制   → S200出草案 → Genius2推演修正 → S200应用偏移后采样输出
Genius2不生成文本 → 只对概率分布施加<2%的偏移
生命周期   → 3-4周/批, 错峰更换
慢性问题   → 信号漂移、单元不可互换、沉默单元、死亡单元
```
