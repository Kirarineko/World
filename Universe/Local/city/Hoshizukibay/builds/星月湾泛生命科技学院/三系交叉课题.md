# 三系交叉课题

## 核心课题链：海洋生物信息采集与处理

这是 HBI 三大学部之间最深层的结构性联系——不是某一个项目，而是一条完整的"感知→采集→处理→应用"链路：

```
海科：发现、采样、验证
  ↓  生物样本 + 环境数据
计科：建模、分析、算力
  ↓  序列比对 + 信号解码 + 模拟结果
生科：改造、合成、器件化
  ↓  活体传感器 / 生物基材 / 湿件处理器
海科：放回海里，开始下一轮
```

这个闭环的意义在于：**海科在研究的那个海湾，同时也是生科改造出的生物体的最终验证场**。一只在 C1 培养皿里会发光的工程藻，放到海湾浮标上能不能活过两周？——这是海科用连续观测数据回答生科的问题。

## 当前在研交叉项目

| 项目 | 三系参与 | 简述 |
|------|----------|------|
| 海湾魔力背景场长期监测 | 海科采样 + 计科建模 + 生科感知菌株 | 将生科开发的魔力感知工程菌（绿色荧光）封装在海科的海湾监测浮标中，利用计科的魔力流扩散模型解读监测数据。目的是绘制海湾全年魔力浓度时空分布图——不是为了追问魔力从哪来，只是因为"这数据没人测过" |
| 藻类光电薄膜海水版 | 海科提供海水微藻 + 生科薄膜封装 + 计科效率建模 | 将淡水藻光电薄膜技术移植到海洋微藻上。目标是在海湾浮标上实现能源自持——浮标本身不需要外部供电，靠藻类薄膜发电维持传感器和通讯模块 |
| 湿件环境信号处理器 | 生科神经元网络 + 计科信号解码 + 海科环境场域 | 用湿件神经元网络替代传统电子芯片处理海洋环境传感器信号。逻辑是：海洋环境数据充满噪声，湿件天然擅长在噪声中识别模式——而硅基芯片不擅长 |
| eDNA 种群动力学 | 海科采样 + 计科序列分析 + 生科分子探针设计 | 在现有 eDNA 物种监测基础上，开发针对特定目标物种（如"灯塔蓝"裸鳃类）的超灵敏检测探针。生科负责设计分子信标，计科负责从测序噪音中提取信号 |

## 交叉机制

### 必修跨学科选修

从大二开始，每个学生至少完成两门其他学部的实验课。课程设计刻意不降低难度——海科的学生来生科楼做蛋白质凝胶电泳，计科的学生上船做浮游生物拖网，该翻车的都会翻车。这条规定在课程评价表上常年被骂，但毕业生的回顾问卷里它常年排第一。

### 交叉课题工作室

大三学生可以申请进入"交叉课题工作室"——一个设在 C5 的开放式项目空间。工作室由三名来自不同学部的教授联合指导，每周一次集体汇报，寒假不休。进入条件只有一条：你的课题必须涉及至少两个学部。课题选题自由——曾有一个学生的毕设是把菌丝体培养在渔船船底的防污涂层上，看看菌丝能不能吃掉附着的藤壶幼体。结论是不能，但菌丝在船底环境下活下来了——这不是一个成功的技术方案，但作为一个科学问题，它有答案。

### 海湾数据共享池

海科学部三十年的连续观测数据、计科学到数据学到的处理管线、生科的生物响应测试数据——三者在校园内网上的同一个数据库里。任何 HBI 师生都可以访问。这不是某个人的远见，是各取所需：海科需要有人帮他们分析数据，计科需要真实数据训练模型，生科需要知道他们改造的生物在海湾真实环境里有多惨。

> 共享池的 FAQ 页有一条："数据可以用于课程作业吗？——可以。可以用于发论文吗？——可以，但要把参与数据采集的学生列在致谢里。可以用于校外商业用途吗？——不能。可以让我的深度学习模型在数据集上刷榜吗？——这又不是 ImageNet，你清醒一点。"

## 不在进行的交叉

有些组合没有出现在 HBI 的课题列表里，不是因为不好，是因为客观条件不够：

| 方向 | 瓶颈 |
|------|------|
| 魔力-海洋生态交叉 | 海水中魔力背景浓度太低（亚空间渗透率低），信号淹没在噪声里 |
| 深海生物采样 | HBI 没有深潜设备，海月号的钢缆放到底也就 200 米 |
| 湿件在海水中的长期稳定性 | 神经元培养皿和海水之间隔着一层膜——膜的寿命决定了湿件在海里的寿命。目前最好的膜撑不过 48 小时 |
| 魔力-神经接口走向临床应用 | 伦理委员会不可能批。HBI 对此有一个内部结论：坚持体外实验，不做动物实验，不碰活人 |

## 速查

```
课题链：海科采样 → 计科建模 → 生科改造 → 海科验证（闭环）
核心项目：魔力背景场监测、藻类光伏浮标、湿件信号处理、eDNA探针
交叉机制：必修跨学科选修 + C5工作室 + 海湾数据共享池
不在进行：魔力-海洋生态（信号太弱）、深海（没设备）、湿件入海（膜寿命）、神经接口临床（伦理）
```
